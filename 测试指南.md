# 🤖 双足机器人步行测试指南

> 发给现场测试人员的操作手册。

---

## 准备工作

### 硬件清单
- [ ] 双足机器人下半身 (10个舵机, ID 1~10)
- [ ] 7.4V 2S 锂电池 (充满电)
- [ ] USB 转串口模块 (CH343/CH340)
- [ ] USB 线连接电脑

### 软件环境
```bash
# 安装 Python 依赖 (只需运行一次)
pip install pyserial
```

### 接线
```
电脑 USB ──→ USB转串口模块 ──→ 舵机信号线
电池 7.4V ──→ 舵机电源线
⚠️ 串口模块 GND 和电池 GND 必须共地!
```

---

## 运行测试

```bash
# 进入项目目录
cd C:\GitHub\servo_robot_temp

# 运行完整测试 (把 COM3 改成你的实际串口号)
python tools/test_walk.py --port COM3
```

> **不知道串口号?** 打开 设备管理器 → 端口(COM和LPT)，找到 CH343 或 USB Serial。

---

## 测试流程 (共8步)

| 步骤 | 内容 | 预计时间 | 说明 |
|:---:|------|:---:|------|
| 0 | 环境检查 | 10秒 | 自动检查 Python/串口 |
| 1 | 连接舵机 | 10秒 | 确认串口线和电源 |
| 2 | 扫描舵机 | 20秒 | 应发现 10 个舵机在线 |
| 3 | 零位站立 | 30秒 | 机器人应直立站好 |
| **4** | **逐关节验证** | **5分钟** | **最重要! 逐个确认方向** |
| 5 | IK 联合测试 | 1分钟 | 左腿多关节同时运动 |
| 6 | 小幅步行 | 30秒 | 保守参数试走 |
| 7 | 正常步行 | 30秒 | 正式步行测试 |

---

## ⚠️ 安全须知

1. **全程用手扶住机器人** — 开环行走没有平衡控制，会倒
2. **异常立刻按 Ctrl+C** — 脚本会安全停止并回到站立
3. **电池电量** — 低于 6.5V 停止测试，防止过放
4. **舵机发烫** — 如果某个舵机很热，暂停休息 5 分钟

---

## 遇到问题时

### 问题1: 找不到串口
→ 检查 USB 线是否插好，驱动是否安装 (CH343 驱动)

### 问题2: 舵机扫描不到
→ 检查电源开关、电池电量、信号线连接

### 问题3: 零位站立姿态不对
→ 脚本会自动读取实际位置，把显示的数据截图发回

### 问题4: 关节方向反了 (Step 4)
→ 这是**最关键的信息**！记录哪个关节反了，发回给开发者

### 问题5: 步行效果不好
→ 描述具体现象 (抖动/倒向哪边/腿没抬起来等)

脚本会自动生成一个测试报告文件:
```
servo_robot_temp/test_report_YYYYMMDD_HHMMSS.json
```

**📬 请把这个文件发回给开发者！**

它包含了所有测试结果和你的反馈，开发者可以据此远程调整参数。

## 如果你想手动调参数

测试完成后，可以用 `walk.py` 单独调参:

```bash
# 更慢更稳的参数
python tools/walk.py --port COM3 --stride 10 --lift 8 --sway 5 --period 1.8

# 更快更大步
python tools/walk.py --port COM3 --stride 30 --lift 20 --sway 10 --period 1.0
```

| 参数 | 说明 | 建议范围 |
|------|------|---------|
| `--stride` | 步幅 (mm) | 5~40 |
| `--lift` | 抬脚高度 (mm) | 5~30 |
| `--sway` | 侧摆 (mm) | 3~15 |
| `--period` | 步态周期 (秒) | 0.8~2.0 |
| `--steps` | 步数 | 2~20 |
